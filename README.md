# Contract-OS — 세움디자인하우징 전산 계약서

수기로 작성하던 ㈜세움디자인하우징 계약서를 **전산 계약서**로 만든 웹앱입니다.
원본 양식 그대로 화면에서 작성하고, 금액을 자동 계산하며, 인쇄/PDF로 출력하고, 계약 건을 저장·관리할 수 있습니다.

## 주요 기능

- **원본 양식 그대로 작성** — 계약금액 · 주문내용(8개 항목) · 약관 · 공급자/계약자 서명란을 동일한 레이아웃으로 화면에서 입력
- **전자 서명** — 서명란의 `✎ 서명` 버튼을 누르면 서명 패드가 열려 마우스·터치·스타일러스로 직접 서명. 서명은 계약 데이터에 함께 저장되고 인쇄/PDF에도 그대로 출력 (미서명란은 인쇄 시 기존 `(인)` 표시 유지)
- **금액 자동계산**
  - 평당 항목은 `평수 × 단가`로 항목 금액 자동 계산
  - 제품공급가(항목 합계) · 부가세(10%) · 제품합계 자동 산출
  - 결제 스케줄(계약금·중도금1·중도금2·잔금)을 제품합계 기준 비율로 **자동 배분** — 계약금 10%(계약 시) · 중도금1 40%(기초공사 완료 후) · 중도금2 45%(골조·외장·지붕 공사 완료 후) · 잔금 5%(나머지, 준공/출고 전). 계약금·중도금은 백만원 단위로 내림하고 끝자리는 잔금이 흡수
  - 고객 요청 시 결제 금액을 칸에 **직접 입력해 수동 조정** 가능(자동 배분 중지). `자동 배분으로 되돌리기` 버튼으로 다시 비율 배분으로 복귀 — 합계가 제품합계와 일치하는지 실시간 안내, 한글 금액 표기
- **인쇄 / PDF 출력** — `인쇄` 버튼 → A4 가로 한 장. 입력칸 배경·테두리가 사라져 인쇄본은 깔끔한 계약서로 출력 (브라우저 인쇄에서 "PDF로 저장" 선택 시 PDF 생성)
- **계약 건 저장·관리** — 계약마다 연도별 번호(예: `2026-0001`) 자동 부여, 목록에서 건축주명·현장주소·계약번호 검색, 불러오기·수정·삭제, 작성중/확정 상태 표시

## 실행 방법

사전 준비: **Node.js 22.5 이상** (외부 라이브러리 설치 불필요 — Node 내장 기능만 사용)

```bash
npm start          # 또는: node --no-warnings server.js
```

브라우저에서 <http://localhost:3000> 접속.

> 포트를 바꾸려면: `PORT=8080 npm start`

## 배포 (Netlify)

이 저장소는 Netlify 배포에 맞춰 구성돼 있습니다.

- 프론트엔드(화면)는 `public/` 폴더가 정적 호스팅됩니다.
- 저장 API는 **Netlify Functions**(`netlify/functions/api.mjs`)로 동작하며, 데이터는 **Supabase**(PostgreSQL)에 저장됩니다.
- 설정은 `netlify.toml`에 들어 있어, Netlify에 저장소를 연결하면 자동으로 빌드·배포됩니다.

> Netlify는 상시 서버를 실행하지 못하므로, 로컬용 `server.js`(SQLite) 대신 배포 환경에서는 Functions + Supabase가 같은 REST API를 제공합니다. 프론트엔드 코드는 동일하게 동작합니다.

### Supabase 설정

1. Supabase 프로젝트의 **SQL Editor**에서 `supabase/schema.sql`을 실행해 `contracts` 테이블을 만듭니다.
2. Netlify 대시보드 → **Site settings → Environment variables**에 아래 두 값을 추가합니다.
   - `SUPABASE_URL` — 프로젝트 URL (예: `https://xxxx.supabase.co`)
   - `SUPABASE_SERVICE_ROLE_KEY` — 프로젝트의 **service_role** 키 (서버 전용, 절대 공개·커밋 금지)
3. 환경변수를 저장한 뒤 **재배포(Deploy)** 하면 적용됩니다.
4. Netlify 대시보드에서 **Production branch가 배포할 브랜치(main)** 로 설정돼 있는지 확인하세요.

### 로그인(직원 인증) 설정

직원이 로그인해서 **자기가 만든 계약만** 보도록 하려면 아래 환경변수를 추가합니다. (세움os와 **같은 Supabase 프로젝트**를 쓰면 세움os 직원 계정 그대로 로그인됩니다.)

- `SUPABASE_ANON_KEY` — 프로젝트의 **anon(public)** 키. 브라우저 로그인용 공개 키라 노출돼도 안전합니다.
  **이 값을 설정하면 로그인 필수 모드가 켜집니다.** (설정 안 하면 기존처럼 로그인 없이 개방)
- `ADMIN_EMAILS` — **전체 계약**을 볼 수 있는 관리자 이메일. 쉼표로 여러 명 지정 가능. (예: `ceo@seum.com,office@seum.com`)

동작:
- 관리자(위 이메일)는 **모든 계약**을 열람·수정합니다.
- 그 외 직원은 **본인이 만든 계약만** 보입니다. (계약 생성 시 로그인 사용자가 소유자로 자동 기록)
- 로그인 이전에 만들어진 **기존 계약**은 소유자 기록이 없어 관리자에게만 보입니다.
  단, 계약의 `영업사원`명이 로그인 사용자 이름과 같으면 그 직원에게도 보이도록 처리했습니다(전환기 배려).
  기존 계약을 특정 직원에게 확실히 귀속시키려면 관리자가 열어서 저장(수정)하면 소유자가 채워집니다.

> 로그인은 외부 라이브러리 없이 Supabase Auth REST API를 직접 호출합니다. 액세스 토큰은 브라우저 `localStorage`에 보관되고, 서버(Functions)가 매 요청의 토큰을 검증해 권한을 강제합니다.

## 데이터 저장 / 백업

- 모든 계약 데이터는 `data/contracts.db` (SQLite) 파일 하나에 저장됩니다.
- **백업**: `data/` 폴더의 `.db` 파일을 복사해 두면 됩니다.
- 이 파일은 git에 포함되지 않습니다(`.gitignore`).

## 폴더 구조

```
Contract-OS/
├── server.js          # HTTP 서버 + REST API (Node 내장 http)
├── lib/db.js          # 저장소 (Node 내장 node:sqlite)
├── public/
│   ├── index.html
│   ├── css/app.css    # 화면 + 계약서 레이아웃
│   ├── css/print.css  # 인쇄/PDF (A4) 레이아웃
│   └── js/
│       ├── model.js   # 계약 데이터 기본값(항목·단가·약관) + 자동계산
│       ├── auth.js    # 로그인(Supabase Auth) + 토큰 관리
│       ├── api.js     # 서버 통신
│       ├── sign.js    # 전자 서명 패드 (캔버스)
│       └── app.js     # 목록/작성 화면, 자동계산, 저장/인쇄
└── data/              # 계약 DB 저장 위치 (백업 대상)
```

## API (참고)

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET    | `/api/contracts?q=검색어` | 목록 / 검색 |
| POST   | `/api/contracts` | 새 계약 생성(번호 자동 부여) |
| GET    | `/api/contracts/:id` | 단건 조회 |
| PUT    | `/api/contracts/:id` | 수정 |
| DELETE | `/api/contracts/:id` | 삭제 |
